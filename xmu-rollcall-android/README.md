# XMU Rollcall Android App

本项目是 [XMU 微信签到机器人](..) 服务的原生 Android 客户端移植。它将原来运行在服务端的 Python 脚本、缓存存储以及微信命令行对话指令，重构为运行在安卓手机本地的原生 App，并提供 Compose 图形界面。

当前 Android 版的推荐工作流是：后台守护负责定时轮询和通知提醒；用户收到提醒后进入 App，按需点击“一键签到”主动提交数字签到或雷达签到。

## 核心特性

1. **完全本地运行**：无需部署 Linux 服务器，无需配置 Python 环境，直接在手机上登录和处理签到。
2. **多账号管理**：支持添加多个学号和密码，密码使用 Android KeyStore 进行硬件级的安全加密存储。
3. **查询与提交解耦**：点击“查询签到”只会检查当前活动签到，并展示签到类型、状态和数字签到码，不会提交签到。
4. **即时一键签到**：点击“一键签到”后，App 会按签到类型主动发送请求：
   - 数字签到：获取 `number_code` 后提交数字签到请求。
   - 雷达签到：根据接口返回的距离信息求解可用坐标后提交雷达签到请求。
   - 二维码签到：当前只识别为不支持类型，不会自动提交。
5. **后台守护模式 (Watch/Cron)**：开启后台守护后，应用会启动前台服务 (Foreground Service) 定时轮询签到。
   - 默认行为是只在检测到新签到时发送状态栏通知，不会直接提交。
   - 如手动开启“检测到签到时自动提交”开关，后台守护会在检测到签到后尝试提交并发送结果通知。
   - 自动提交后会复查 TronClass 返回的签到状态，只有确认完成后才显示成功。
   - 轮询间隔可在 App 内通过滑块设置，当前范围为 1 到 15 分钟。

## 推荐使用方式

1. 首次打开 App，授予通知权限。
2. 在“账号设置”中添加 XMU 统一身份认证账号。
3. 打开“后台自动扫描守候”。
4. 保持“检测到签到时自动提交”为关闭状态，以便后台只提醒、不提交。
5. 收到“检测到新签到”通知后，点击通知进入 App。
6. 确认签到内容后，点击“一键签到”提交当前活动签到。

如果你希望完全自动处理，也可以开启“检测到签到时自动提交”。开启后，后台守护会把检测和提交合并执行。

## 功能边界

- 支持：账号本地加密保存、移动端 H5 登录、定时轮询、通知提醒、手动一键处理数字签到和雷达签到。
- 不支持：二维码签到自动提交。
- 注意：后台守护已使用前台服务、WakeLock 和开机恢复来提高稳定性，但仍依赖 Android 与厂商 ROM 的后台策略，强省电 ROM 仍可能冻结或杀死进程。

## 快速开发与编译指南

### 1. 编译环境
* **Android Studio** (推荐 Jellyfish 或更高版本)
* **JDK 17** (Android Gradle Plugin 8.2 的标准运行环境，Android Studio 会自动附带)
* 支持的安卓版本：Android 8.0 (API 26) 及以上

### 2. 构建与运行
1. 打开 **Android Studio**。
2. 选择 **Open**，导航并选中 `xmu-rollcall-android` 文件夹导入项目。
3. 等待 Gradle 自动下载并完成项目同步 (Sync)。
4. 连接您的 Android 手机（需开启 USB 调试模式）或启动内置模拟器。
5. 点击顶部的 **Run** 按钮（绿色三角形）将 App 安装至设备。

### 3. 生成 Release APK
在 Android Studio 菜单中选择 `Build` -> `Build Bundle(s) / APK(s)` -> `Build APK(s)`，编译成功后即可获取安装用的 `.apk` 文件。

> 备注：当前仓库未提交 `gradlew` / `gradlew.bat` 和 `gradle-wrapper.jar`，命令行构建需要本机已安装 Gradle，或先在 Android Studio 中同步项目。

## 后台保活说明

由于 Android 系统的墓碑机制和省电策略非常激进，后台运行的扫描守护可能会在手机休眠时被系统冻结或杀死。为了确保您在上课期间不漏过任何签到，建议进行以下系统设置：

1. **授予通知权限**：App 启动时会请求发送通知的权限，请务必同意（以展示常驻服务通知和签到提醒）。
2. **忽略电池优化**：
   - 前往手机的 `系统设置` -> `应用管理` -> `XMU 签到助手` -> `电池/省电策略`。
   - 将省电策略从“智能限制”或“限制后台行为”更改为 **“无限制” (Unrestricted)**。
3. **锁定多任务后台**：
   - 打开手机的最近任务列表 (Recent Apps)。
   - 找到本应用，下滑卡片并点击“锁定” (Lock) 图标，防止一键清理时被强制关闭。
